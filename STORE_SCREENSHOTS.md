# Chrome Web Store screenshot checklist

Capture five real extension screenshots at exactly 1280 × 800 pixels. Use a
regular public web page with visually strong, non-sensitive images. Keep the
Block Poster side panel open and make sure no personal email address, browser
profile menu, unrelated tabs, notifications, or private page content appears.

## 1. Pick an image from the page

- Show the page and the Block Poster side panel together.
- Show a selected image in the extension preview.
- Keep the source actions and compact monochrome interface visible.

## 2. Capture a page region

- Show the live region-selection overlay on a regular website.
- Make the selected rectangle large and immediately understandable.
- Avoid selecting copyrighted or sensitive text as the main subject.

## 3. Scan and select a batch

- Show a page scan with a clean image grid.
- Select several images so the checkboxes and ZIP action are visible.
- If the account badge is shown, use a Pro or Lifetime state without exposing
  an email address.

## 4. Configure the print layout

- Show one selected image, Print setup, Pages across, the calculated poster
  layout, Margin, Overlap, and the Cut marks & page labels setting.
- Use values that produce a practical grid rather than the maximum 8-page
  width.

## 5. Export choices

- Show the sticky action bar with both Free PDF/ZIP and Watermark-free PDF/ZIP.
- Prefer a state where the selected image preview and action bar are visible in
  the same frame.
- Do not show a payment error, loading state, toast, or permission warning.

## Image rules

- PNG or JPEG; 1280 × 800 preferred by Chrome Web Store.
- Square corners and full bleed; do not add a decorative browser frame.
- Use the current extension version only.
- Keep text minimal and readable after the store downsizes the image.
- Do not add “Featured”, “Editor’s Choice”, rankings, reviews, or unsupported
  claims.
