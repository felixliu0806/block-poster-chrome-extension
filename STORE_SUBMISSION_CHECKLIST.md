# Chrome Web Store submission checklist

## Files prepared in this repository

- Upload package: `release/block-poster-0.1.0.zip`
- Store icon: `store-assets/icon-128.png`
- Small promo tile: `store-assets/promo-small.png`
- Marquee promo tile: `store-assets/promo-marquee.png`
- Five upload-ready screenshots: `store-assets/screenshots/`
- Listing copy and permission explanations: `STORE_LISTING.md`
- Screenshot plan: `STORE_SCREENSHOTS.md`

## Developer account

- Register the publisher account and complete the one-time developer fee.
- Enable 2-Step Verification on the publishing Google account.
- Complete publisher identity verification if the Dashboard requests it.
- Verify `blockposter.pro` in the same publisher account so it can be selected
  as the Official URL.

## Store listing

- Name: Block Poster
- Primary language: English
- Category: Art & Design
- Copy the summary and detailed description from `STORE_LISTING.md`.
- Homepage: `https://blockposter.pro/`
- Support: `https://blockposter.pro/support`
- Privacy policy: `https://blockposter.pro/privacy-policy`
- Upload the icon, five real screenshots, small promo tile, and marquee tile.
- Declare that the extension uses in-app purchases because clean exports and
  larger watermark-free batches are purchased on `blockposter.pro`.

## Privacy tab

- Single purpose: Turn user-selected web images and screen captures into
  printable tiled poster PDFs.
- Disclose Website content.
- Disclose Authentication information because a scoped account token is stored
  and transmitted to Block Poster APIs.
- Certify that data is not sold, used for personalized advertising or credit
  decisions, or used outside the extension's user-facing purpose.
- Use `STORE_LISTING.md` for the detailed permission justifications.

## Distribution and review

- Choose Public when the listing and screenshots are final.
- Select all regions unless there is a specific legal or support reason not to.
- Save the item as a draft first and review every warning before submitting.
- Do not claim “Featured”, “Editor’s Choice”, rankings, or review scores.
- After publication, replace “Chrome Web Store release in progress” on the
  website landing page with the real store link and official store badge.
